## 📤 How to Manage Friend Contributions

### **When Friend Wants to Contribute:**

#### **1. Add Them as Collaborator** (Optional)
In your GitHub repo:
- Go to **Settings** → **Collaborators**
- Click **Add people**
- Enter their GitHub username
- They get notification to accept

#### **2. Show Them How to Contribute**

Send them this workflow:

```bash
# 1. Make sure they're up to date
git pull origin main

# 2. Create a feature branch
git checkout -b feature/their-feature-name

# 3. Make their changes
# (Edit files, test locally)

# 4. Commit changes
git add .
git commit -m "Description of what changed"

# 5. Push to GitHub
git push origin feature/their-feature-name

# 6. Create Pull Request on GitHub
# (They'll see a prompt on GitHub to create PR)
```

#### **3. Review Their Code**
- Go to **Pull Requests** tab on GitHub
- Click their PR
- Review changes
- Click **Merge pull request** if good
- Click **Confirm merge**

#### **4. Pull Latest Changes**
Back in your local project:
```bash
git pull origin main
```

## 🆘 Common Questions Friends Will Ask

### **Q: "Can I see the code without running it?"**
A: Yes! Visit the GitHub repo and browse files in the browser.

### **Q: "Do I need to pay for MongoDB?"**
A: No! Free tier (M0) is included in MongoDB Atlas.

### **Q: "Do I need to buy an API key?"**
A: No! OpenWeatherMap has a free tier with 1000 calls/day.

### **Q: "Can I push my changes directly?"**
A: Only if you add them as collaborators. Otherwise, they create a Pull Request.

### **Q: "What if I break something?"**
A: No problem! GitHub tracks all changes. You can revert to previous versions.

### **Q: "How do I update my local code when you push updates?"**
A: They run: `git pull origin main`
