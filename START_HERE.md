## 🚀 YOUR 3-STEP GITHUB SETUP (30 MINUTES)

### **STEP 1: Prepare Git (5 minutes)**
Open Command Prompt/PowerShell in `c:\trip-plan`:
```bash
git init
git config user.name "Your Name"
git config user.email "your-email@gmail.com"
git add .
git commit -m "Initial commit: Smart Tourism Assistant"
```

### **STEP 2: Create GitHub Repo (3 minutes)**
1. Go to https://github.com/new
2. Repository name: `trip-plan`
3. Description: "AI trip planner with 943 hidden gems"
4. Visibility: **PUBLIC** (so friends can see it)
5. Click **Create repository**

### **STEP 3: Push Your Code (7 minutes)**
Back in Command Prompt:
```bash
git remote add origin https://github.com/YOUR_USERNAME/trip-plan.git
git branch -M main
git push -u origin main
```

**When asked for password:**
- Use your **Personal Access Token** (see below how to create)

### **STEP 3B: Create Personal Access Token (5 minutes)**
1. Go to GitHub → Profile photo → **Settings**
2. **Developer settings** (left menu) → **Personal access tokens** → **Tokens (classic)**
3. **Generate new token (classic)**
4. Name it: "trip-plan"
5. Check: `repo` checkbox
6. Click **Generate token**
7. **Copy it** (you won't see it again!)
8. Use this as your password when pushing

## 🆘 IF YOU GET STUCK

| Problem | Solution |
|---------|----------|
| "Git not found" | Download from git-scm.com |
| "Can't push" | Generate Personal Access Token (5-min process) |
| "Push fails" | Check repo is public not private |
| "Friend can't clone" | Check link is correct + repo is public |
| "Hidden gems don't show" | Run `npm run seed` in server folder |
| "MongoDB won't connect" | Check MONGO_URI in .env |
| "Port already in use" | Change PORT in .env to 5001 or 5002 |

**For detailed help:** Read `HIDDEN_GEMS_TROUBLESHOOTING.md`
